## 🎯 模板结构标准

每个模板包含：

```markdown
### 模板名称

**场景**: 使用场景描述

**难度**: ⭐⭐⭐

**原始提示**:
[用户可能写的烂提示]

**优化后提示**:
```xml
<instructions>
[结构化提示词]
</instructions>
```

**为什么有效**:
50-100 字科学解释

**替换区**:
🔴 这里改你的产品名
🔵 这里改你的目标受众
🟢 这里改你的具体要求

**一键复制**: [按钮]
**一键优化**: [按钮]
```

## 🦞 特色功能规划

### 1. 坏提示翻译器
```
输入："帮我写个文案"
↓
输出："你是一位专业文案策划，请为 [产品] 创作 [平台] 风格的文案..."
```

### 2. 提示词体检报告
```
粘贴你的提示词
↓
得分：65/100
问题：缺少角色定义、无输出格式
建议：添加<instructions>标签...
```

### 3. 模型口味切换
```
同一需求 → Claude 版 / Qwen 版 / Gemini 版
```
